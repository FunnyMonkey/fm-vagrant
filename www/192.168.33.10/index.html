You are currently accessing this site via direct 192.168.33.10.

Dynamic virtual host aliases

You can also host via dynamic sub-domains just create a folder with the appropriate name. For example if you add a folder 'fm-vagrant.local' to the 'www' directory in this repo and you put an entry in your 'etc/hosts' file mapping 192.168.33.10 to fm-vagrant.local then you will serve the files located in fm-vagrant.local directory when you access fm-vagrant.local.
